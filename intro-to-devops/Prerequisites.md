# Absolute Prerequisite for Learning DevOps

\---

## Goal

Understand how software teams work before learning DevOps tools and practices.

\---

## 1\. Roles in a Company

### Business Analyst (BA)

* Talks with customers
* Collects requirements
* Prepares BRD (Business Requirement Document)

### Product Manager (PM)

* Reviews BRD
* Prioritizes features based on business needs and team capability

### Product Owner (PO)

* Breaks features into smaller actionable items called Epics
* Makes requirements clear for the team

### Software / Solutions Architect

* Creates technical design (HLD / LLD)
* Decides if a feature is technically possible

### Developers

* Build the required features

### QE Engineers

* Test the application and verify quality

### Database Administrator (DBA)

* Manages and monitors the database

### DevOps Engineer

* Builds CI/CD pipelines
* Manages deployment environments

### Site Reliability Engineer (SRE)

* Monitors reliability, dashboards, alerts, and performance

### Technical Writers

* Create technical and user documentation

\---

## Scrum Basics

### Scrum

An Agile framework where teams work together in short cycles called Sprints.

### Sprint

* Usually lasts 2–3 weeks
* Focuses on completing planned tasks

### Sprint Retrospective

* Reviews progress
* Discusses improvements for the next sprint

### Example Scrum Team

* Developers
* DevOps
* DBA
* QE Engineers
* Other supporting roles

\---

## 2\. SDLC (Software Development Life Cycle)

### 1\. Requirement Gathering

Business Analyst collects requirements

### 2\. Planning

Product Manager prioritizes the work

### 3\. Design

Product Owner and Architect prepare technical design

### 4\. Development

Developers build the feature

### 5\. Testing

QE Engineers test the system

### 6\. Deployment

DevOps deploys using CI/CD pipelines

### 7\. Monitoring

SRE and DevOps monitor performance and reliability

\---

## 3\. Jira Environment

### Product Owner

* Creates and breaks down features
* Assigns work to the Scrum Team

### Scrum Team

* Works on assigned tasks
* Updates task status:

  * To Do
  * In Progress
  * Testing
  * Done

\---

## Work Types

### Epic

Large feature or business requirement

Example:

* User Authentication System

### Story

Smaller requirement inside an Epic

Example:

* Login Page
* Forgot Password

### Task

Specific technical work

Example:

* Create login API

### Bug

Issue or defect to be fixed

Example:

* Login button not working

\---

## Final Note

DevOps is not only about tools like Docker, Kubernetes, Jenkins, or AWS.

It is about improving how teams build, deploy, and maintain software efficiently.
