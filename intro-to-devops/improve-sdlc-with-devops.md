# Software Development Life Cycle (SDLC)

## What is SDLC?

SDLC (Software Development Life Cycle) is a standard process used in the software industry to ensure that requirements are delivered with high quality.

Everyone working in a software company should understand this process.

Its goal is to build software in a structured, efficient, and reliable way.

## Stages of SDLC

### 1. Planning

- Gather business requirements
- Understand what the customer needs
- Business Analyst and Product Manager are usually involved

### 2. Defining

- Convert requirements into proper documentation

Example:

- BRD (Business Requirement Document)
- Functional requirements
- Technical requirements

This helps the team clearly understand what needs to be built.

### 3. Designing

- Create the technical structure of the system

### HLD (High-Level Design)

Usually handled by senior-level engineers or architects

Includes:

- System architecture
- Service structure
- Database design
- Overall technical flow

### LLD (Low-Level Design)

Usually handled by developers or junior-level engineers

Includes:

- Functions
- Tools to be used
- API details
- Implementation logic

## DevOps Starts Here

DevOps is highly involved from Designing to Deployment to make delivery faster and more reliable.

### 4. Building

- Developers write the code
- Use Git for version control
- Push code to GitHub or repository platforms

### 5. Testing

- Ensure the application works correctly
- QE (Quality Engineers) test the system
- Validate features before production release

The goal is to prove that the application is working properly.

### 6. Deployment

- Promote the application to production
- Customers can now use the application

This is where DevOps helps heavily through CI/CD pipelines.

### 7. Monitoring and Maintenance

- Monitor system performance after deployment
- Track errors, downtime, and reliability
- Fix bugs and improve performance when needed

SRE and DevOps teams are commonly involved here.

Even if some videos only mention 6 stages, monitoring is very important in real-world projects.

## Agile Method

Most companies use the Agile method.

This means:

- Teams work in smaller cycles called Sprints
- Some stages can start without waiting for the previous stage to fully finish
- Faster delivery and continuous improvement become possible

Unlike traditional models, Agile allows teams to work in parallel.

## Role of DevOps in SDLC

DevOps Engineers help speed up the SDLC process by:

- Automating repetitive tasks
- Building CI/CD pipelines
- Reducing manual errors
- Improving deployment speed
- Monitoring application reliability

Their goal is to make software delivery more efficient and stable.

## Final Note

DevOps does not replace SDLC.

DevOps improves SDLC by making each stage faster, smoother, and more reliable through automation and collaboration.
